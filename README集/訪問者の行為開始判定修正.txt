era紅魔館protoNTR用
	「一定の条件下で訪問者と住人が行動できなくなる現象の改善」

パッチの簡単な説明
	訪問者が一定の素質持ちの住人と同じ場所にいると行動できなくなる不具合の応急処置

	ERBフォルダを入れてください

〜パッチ内容〜
	何が原因で行動ができなくなるのかという部分は不明ですが、どういった処理によって行動が行えなくなるかが判明したので
	行動が行えなくなる現象改善への応急処置として修正を加えました

	SELECTCASEにおいて何らかの原因で訪問者のムードがCASEにない状態になっている？
	お持ち帰りがONになっていなければ起こらないことから
	「NTR_MOOD_性交」以上「NTR_MOOD_同伴」以下の何かになっている可能性があります

	訪問者のムードか他の数値が影響してCASEにない状態になったため
	「どのケースも選べない→行動を行えない→ムード上昇も起こらない」となり行動不能になっていたものと思われます
	よってCASEELSEを設定し、CASEELSEになった場合は強制的に会話を行う処理へGOTOさせることで行動不能状態を解除し、
	ムードを上げさせることでCASE NTR_MOOD_同伴を選択できる状態にするという処理にしました
